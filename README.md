# Chrome Profiles Manager

Менеджер Chrome-профилей для Windows с синхронизацией действий: назначаете одно окно «мастером» — клики, клавиатура и навигация зеркалируются во все остальные профили.

## 🚀 Возможности

- **Управление профилями** — создание, удаление, запуск и остановка изолированных Chrome-профилей из GUI (PyQt). Каждый профиль — отдельная папка в `profiles/`.
- **Синхронизация ввода** — глобальные хуки клавиатуры/мыши Windows: действия в мастер-окне повторяются во вторичных окнах (клики, колесо, базовые клавиши).
- **Зеркалирование внутри страниц** — MV3-расширение подключается к локальному WebSocket-хабу (`ws://127.0.0.1:8765`) и повторяет ввод текста и навигацию между профилями.
- **Привязка окон** — автоматическое сопоставление PID/HWND запущенных окон Chrome с профилями.

## 📦 Установка

Требования: **Windows 10/11**, **Python 3.11+**, запуск от администратора (нужен для глобальных хуков).

```bat
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
```

Или просто запустите `install.bat`.

## ▶️ Запуск

```bat
.venv\Scripts\python src\main.py
```

Или `start.bat`. Параметры по умолчанию — в `src/app/config.py`.

## 🧩 Расширение для зеркалирования страниц

1. Откройте `chrome://extensions`, включите **Developer mode**.
2. **Load unpacked** → выберите папку `extension/`.
3. В **Extension options** укажите имя профиля (как в GUI) и WS URL: `ws://127.0.0.1:8765`.
4. Повторите для каждого профиля — настройки хранятся внутри профиля.
5. В GUI назначьте мастер-профиль и откройте сайт в каждой вкладке.

Подробнее: [extension/README_extension.md](extension/README_extension.md)

## 🗂 Структура

```
src/app/
├── gui.py                    # главное окно (PyQt)
├── profile_manager.py        # создание/запуск/остановка профилей
├── chrome_window_manager.py  # поиск и привязка окон Chrome
├── sync_manager.py           # хуки ввода и доставка событий в окна
├── ws_hub.py                 # WebSocket-хаб для расширения
└── config.py                 # настройки
extension/                    # MV3-расширение для зеркалирования страниц
```

## ⚠️ Замечания

- WebSocket-хаб работает только локально — не пробрасывайте его наружу.
- Для хуков и отправки сообщений в окна нужны права администратора.
